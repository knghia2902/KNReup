# Phase 3.1: Tách luồng Pipeline thành Analyze và Render

## Objective
Tách quy trình 1-click hiện tại thành 2 nửa riêng biệt: Analyze (Transcribe + Translate) và Render (TTS + Merge) để người dùng có thể Review/Sửa subtitle.

## Wave 1: Backend Refactor

<task>
<objective>Chia nhỏ luồng PipelineRunner và tạo API Analyze/Render</objective>

<read_first>
- python-sidecar/app/pipeline_runner.py
- python-sidecar/app/routes/pipeline.py
</read_first>

<action>
1. Trong `python-sidecar/app/pipeline_runner.py`:
   - Đổi tên class `PipelineRunner` thành `PipelineRunner` (giữ nguyên tên), nhưng tách hàm `run()` thành `run_analyze()` và `run_render()`.
   - `run_analyze(video_path, config, output_dir)`: 
     - Giữ nguyên luồng Transcribe và Translate.
     - Cuối hàm thay vì chạy TTS, hãy `yield {"stage": "done", "progress": 100, "message": "Analyze complete", "segments": translated_segments}`.
   - `run_render(video_path, config, segments, output_dir)`:
     - Bắt đầu từ luồng TTS (lấy `segments` làm đầu vào), tạo `dubbed_audio_path` và gọi `FFmpegOutputBuilder` để Merge.
     - Cuối hàm `yield {"stage": "done", "progress": 100, "message": "Pipeline complete!", "output_path": output_path}`.

2. Trong `python-sidecar/app/routes/pipeline.py`:
   - Đổi endpoint `/process` thành `/analyze`. Nhận `ProcessRequest`. Gọi `runner.run_analyze()`. SSE trả về nội dung `event`.
   - Thêm class `RenderRequest(ProcessRequest)` có chứa thêm cột `segments: list[dict]`.
   - Tạo endpoint `POST /render` (streaming) nhận `RenderRequest`, gọi `runner.run_render(..., req.segments, ...)`.
</action>

<acceptance_criteria>
- Hàm `run_analyze` trả về event `done` chứa mảng `segments`.
- API route `/api/pipeline/analyze` hoạt động streaming.
- API route `/api/pipeline/render` hoạt động streaming với request có `segments`.
</acceptance_criteria>
</task>
