# Phase 3.1: Context - Tách luồng Pipeline Analyze và Render

**Gathered:** 2026-03-24
**Status:** Ready for planning
**Source:** Tách từ yêu cầu mở rộng Phase 3 (UI Subtitle Editor)

<domain>
## Phase Boundary

Giai đoạn này tập trung vào việc tái cấu trúc `PipelineRunner` phía backend (FastAPI) và `usePipeline` phía frontend (Tauri/React) để thay đổi luồng xử lý 1 chạm (1-click end-to-end) thành 2 giai đoạn độc lập:
1. **Analyze (Phân tích):** Transcribe (Whisper) -> Translate -> Preview trên UI (Cho phép user sửa phụ đề trước khi xuất).
2. **Render (Xuất bản):** Bắt đầu từ UI Subtitle -> TTS (Tạo âm thanh) -> Merge (Ghép âm thanh và phụ đề vào file MP4 cuối bằng FFmpeg).

Người dùng đã chốt:
- Chạy tự động quá trình Analyze ngay khi Import Video.
</domain>

<decisions>
## Implementation Decisions

### Backend Restructuring
- Chia `app/pipeline_runner.py` -> `run_analyze(video_path, config)` và `run_render(video_path, config, segments)`.
- Hàm analyze `yield` progress xong sẽ trả về danh sách `segments` đã dịch hoặc gộp chung.
- Cập nhật endpoints trên `app/routes/pipeline.py` (`/api/pipeline/analyze` và `/api/pipeline/render`).

### Frontend Interactions
- Refactor `src/hooks/usePipeline.ts`: chia hàm `startPipeline` thành `analyzeVideo` và `renderVideo`.
- `App.tsx`: Auto trigger `analyzeVideo` khi có `handleFileSelected` hợp lệ. Lắng nghe tiến trình và cập nhật list Subtitle vào `useSubtitleStore`.
- `SubTab.tsx` / `UploadPanel.tsx` / `OutTab.tsx`: Khi user click Render Output, gọi `renderVideo` truyển mảng segments hiện tại ngược về backend.
</decisions>
