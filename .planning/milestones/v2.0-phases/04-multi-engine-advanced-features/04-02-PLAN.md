# Phase 4 - Plan 02: Audio FX Pipeline & Export Queue

<objective>
Tích hợp khả năng tùy biến Audio FX (Pitch, Speed) qua FFmpeg trên toàn cục Audio Track, hỗ trợ preview nhanh 10 giây. Thêm tính năng cấu hình hàng đợi tự động (Batch Queue) trên giao diện Properties.
</objective>

<tasks>

## Task 1: Extend Pipeline Runner for Audio FX

<action>
In `python-sidecar/app/pipeline_runner.py`, locate `_merge_tts_audio`.
Update the `filter_complex_script` to append an Audio FX chain.
Use `atempo={config.speed}` and `asetrate=44100*{config.pitch}` on the `[out]` node of `amix`. 
Ensure `PipelineConfig` supports `.speed` (float, default 1.0) and `.pitch` (float, default 1.0).
</action>
<read_first>
- python-sidecar/app/pipeline_runner.py
</read_first>
<acceptance_criteria>
- `PipelineConfig` accepts `speed` and `pitch`.
- `_merge_tts_audio` successfully dynamically generates `atempo` and `asetrate` FFmpeg filters and applies them to the final track output.
</acceptance_criteria>

## Task 2: Create Audio FX Preview Endpoint

<action>
Create a new FASTAPI route `POST /api/pipeline/preview-audio` in `python-sidecar/app/routes/pipeline.py`.
The endpoint accepts a short base64 audio sample or path, plus `speed` and `pitch` params.
It runs a synchronous (or fast async) `ffmpeg -i input.mp3 -filter:a "atempo=...,asetrate=..." output.mp3`.
Returns the processed MP3 as a `FileResponse`.
</action>
<read_first>
- python-sidecar/app/routes/pipeline.py
</read_first>
<acceptance_criteria>
- Endpoint `/api/pipeline/preview-audio` exists.
- The route invokes FFmpeg properly and returns an audio file stream.
</acceptance_criteria>
