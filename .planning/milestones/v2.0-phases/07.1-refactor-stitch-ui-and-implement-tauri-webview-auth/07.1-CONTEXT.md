# Phase 07.1: refactor-stitch-ui-and-implement-tauri-webview-auth - Context

**Gathered:** 2026-04-16
**Status:** Ready for planning

<domain>
## Phase Boundary
Implementation of the Tauri WebView Douyin Login feature and the complete design overhaul of the Downloader module UI to meet the Stitch Design System standards (Asymmetric layouts, High-quality typography, Spring-physics motion).
</domain>

<decisions>
## Implementation Decisions

### 1. Tauri WebView Login
- **D-01:** Remove `f2` local browser cookie extraction logic from the UI.
- **D-02:** Create a `DouyinAuthWidget` that opens a Tauri Window (`@tauri-apps/api/window` or `tauri-plugin-oauth`) pointing to `douyin.com`.
- **D-03:** App will intercept and save cookies following successful login in the WebView. UI displays status with spring animation.

### 2. Stitch Design Taste Update
- **D-04:** Use `Outfit` or `Geist` font with `-0.025em` tracking.
- **D-05:** Convert layout from Centered Flexbox to Asymmetric Split Screen (Grid).
- **D-06:** Colors: `#18181B` (text), `#F9FAFB` (bg), `#FFFFFF` (card). Border `rgba(226,232,240,0.5)`. Spring transition `.4s`.
</decisions>
