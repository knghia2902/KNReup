# Plan 07.1-04: Fix Douyin WebView Cookie Capture

## Goal
Fix the COM threading issue in the Python sidecar and automate cookie capture from the Tauri WebView2 window.

## Tasks

### [Task 07.1-04-01] Fix COM threading in `douyin_engine.py`
- Modify `sync_cookie` in `python-sidecar/app/engines/downloader/douyin_engine.py`.
- wrap the browser cookie extraction thread with `pythoncom.CoInitialize()` and `pythoncom.CoUninitialize()`.

### [Task 07.1-04-02] Enable `allow-execute-javascript` in Tauri
- Modify `src-tauri/capabilities/default.json`.
- Add `core:webview:allow-execute-javascript` to permissions.

### [Task 07.1-04-03] Implement Script Injection in `DouyinAuthWidget.tsx`
- Update `src/components/downloader/DouyinAuthWidget.tsx`.
- Use a `setInterval` to periodically `eval` a script in the Douyin WebView that checks `document.cookie` for `sessionid`.
- Automatically call `onSet(cookies)` when captured.

## Verification
- Run app, perform login, verify "Session Active" status is achieved without manual sync from external browser.
