# Phase 6.1 — UI Design Contract (OCR Subtitle Extraction)

## 1. Mục tiêu giao diện
Tạo ra một bộ Box chọn vùng an toàn ngay trên màn hình Preview Video để người dùng trỏ định điểm mà OCR sẽ quét Subtitle, nhằm ngăn chặn việc OCR quét thừa toàn bộ khung hình làm ì ạch CPU.

## 2. Visual Architecture
- **Tab Integration (Properties Panel)**: 
  - Tại file `SubTab.tsx` sẽ mọc thêm một mục Settings mang tên "OCR Extraction".
  - Chứa nút Toggle (Bật/Tắt chế độ chạy OCR).
- **Video Overlay (Player Window)**:
  - Tương tự như cơ chế Blur thủ công vốn có của phần mềm. Áp dụng chung cho `VideoPreview.tsx`.
  - Sẽ xuất hiện một hộp `OCR Region Box` có viền nhận dạng màu **Xanh Lá Cây / Xanh Neon** (phân biệt màu trắng mờ của Blur cũ).
  - Có các `Knob` ở 4 góc cho rê chuột co giãn kích cỡ và kéo rê tọa độ.

## 3. Interactive Elements (Data Flow)
- Giao diện kết nối trực tiếp với file `useProjectStore.ts` của Zustand.
- Cần cung cấp các params:
  ```ts
  ocr_enabled: boolean;
  ocr_x: number;
  ocr_y: number;
  ocr_w: number;
  ocr_h: number;
  ```
- Thao tác kéo thả trên video cần update lên Zustand Store, và truyền xuống backend Python qua param Config dưới JSON string format tự động.

## 4. Constraint & Safety
- **Clamping**: Box OCR không được bao giờ vượt khỏi ranh giới Video Width x Height. (Tính code Scale cẩn thận trong `VideoPreview`).
- Chống giật khung hình: Sử dụng Throttle Dragging Handler khi update data vào Zustand hoặc giữ ref.
