Tạo `src/hooks/useDownloader.ts`:

Custom hook quản lý toàn bộ state của Downloader module qua zustand hoặc React state:

```typescript
interface VideoFormat {
  format_id: string;
  ext: string;
  resolution: string;
  filesize: number | null;
  vcodec: string;
  acodec: string;
  format_note: string;
}

interface VideoInfo {
  title: string;
  uploader: string;
  duration: number;
  thumbnail: string;
  platform: string;
  formats: VideoFormat[];
}

interface DownloadItem {
  id: number;
  url: string;
  platform: string;
  title: string;
  thumbnail_url: string;
  resolution: string;
  file_size: number;
  status: 'pending' | 'analyzing' | 'downloading' | 'completed' | 'error';
  progress: number;
  speed: string;
  error_message: string;
  created_at: string;
  completed_at: string;
}
```

Functions:
- `analyzeURL(url: string)` → POST `/api/download/analyze` → set videoInfo state
- `startDownload(url, format_id)` → POST `/api/download/start` → add to queue, start SSE listener
- `cancelDownload(id)` → POST `/api/download/cancel/{id}`
- `deleteDownload(id)` → DELETE `/api/download/{id}`
- `fetchHistory(limit, offset, platform)` → GET `/api/download/history`
- `syncCookie(browser)` → POST `/api/download/cookie/sync`
- `checkCookie()` → GET `/api/download/cookie/status`
- SSE listener cho download progress via EventSource đọc `/api/download/stream/{id}`

State: `videoInfo`, `isAnalyzing`, `queue: DownloadItem[]`, `history: DownloadItem[]`, `cookieStatus`
</action>

- File chứa `analyzeURL` và `startDownload` và `fetchHistory`
- File chứa `EventSource` hoặc SSE listener code
- File fetch từ `/api/download/`
</acceptance_criteria>
