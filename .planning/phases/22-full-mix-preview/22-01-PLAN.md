# Plan 22-01: Web Audio API Full Mix Preview

<objective>

Tạo file mới `src/lib/audioMixer.ts`:

```typescript
// AudioMixer — singleton quản lý Web Audio API routing graph
// 
// Routing:
//   <video> → MediaElementSourceNode → originalGain → destination
//   TTS AudioBufferSourceNodes → ttsGain → destination
//   <audio> BGM → MediaElementSourceNode → bgmGain → destination
//
// API:
//   AudioMixer.init() — tạo AudioContext, resume
//   AudioMixer.connectVideo(videoEl) — connect <video>, track ref để không double-connect
//   AudioMixer.connectBGM(audioEl) — connect <audio>, track ref
//   AudioMixer.setOriginalVolume(v: number) — set originalGain.gain.value
//   AudioMixer.setTTSVolume(v: number) — set ttsGain.gain.value
//   AudioMixer.setBGMVolume(v: number) — set bgmGain.gain.value
//   AudioMixer.scheduleTTS(segments, startTime) — schedule AudioBufferSourceNodes
//   AudioMixer.cancelTTS() — stop tất cả active source nodes
//   AudioMixer.loadTTSBuffer(path: string): Promise<AudioBuffer> — fetch + decode
//   AudioMixer.dispose() — cleanup
//
// Implementation details:
//   - Track connected elements bằng WeakRef để tránh double-connect (createMediaElementSource chỉ gọi 1 lần per element)
//   - AudioContext lazy init — chỉ tạo khi gọi init() lần đầu
//   - Buffer cache: Map<string, AudioBuffer> — cache decoded TTS buffers
//   - Active TTS source nodes: Set<AudioBufferSourceNode> — để cancel khi seek
```

Key fields:
- `ctx: AudioContext | null`
- `originalGain: GainNode`
- `ttsGain: GainNode`
- `bgmGain: GainNode`
- `connectedVideo: WeakRef<HTMLVideoElement> | null`
- `connectedBGM: WeakRef<HTMLAudioElement> | null`
- `bufferCache: Map<string, AudioBuffer>`
- `activeTTSSources: Set<AudioBufferSourceNode>`

`scheduleTTS` implementation:
1. Cancel existing sources (`activeTTSSources.forEach(s => s.stop())`)
2. Lặp qua segments có `tts_audio_path`
3. Với mỗi segment: load buffer từ cache (hoặc fetch+decode), tạo `AudioBufferSourceNode`, connect tới `ttsGain`
4. Calculate delay: `Math.max(0, segment.start - startTime)` 
5. Calculate offset: `startTime > segment.start ? startTime - segment.start : 0`
6. Call `source.start(ctx.currentTime + delay, offset)`
7. Track trong `activeTTSSources`

`connectVideo` implementation:
1. Check `connectedVideo?.deref() === videoEl` → skip nếu đã connect
2. `const source = ctx.createMediaElementSource(videoEl)`
3. `source.connect(originalGain)`
4. `originalGain.connect(ctx.destination)`
5. `connectedVideo = new WeakRef(videoEl)`

`connectBGM` tương tự nhưng route qua `bgmGain`.
</action>

```typescript
async function preloadTTSBuffers(segments: Array<{tts_audio_path?: string}>) {
  const paths = segments
    .map(s => s.tts_audio_path)
    .filter((p): p is string => !!p);
  
  // Preload unique paths
  const uniquePaths = [...new Set(paths)];
  await Promise.allSettled(
    uniquePaths.map(p => loadTTSBuffer(p))
  );
}
```
