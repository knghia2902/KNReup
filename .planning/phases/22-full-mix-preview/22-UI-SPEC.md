## Component Inventory

### 1. AudioTab Mix Section (Primary Control)

**Location:** Properties panel → Audio tab, new "Mix Preview" section below existing "Original Audio" section.

| Element | Specification |
|---------|--------------|
| Section header | `.pshd` pattern: `SpeakerHigh` icon (16px bold) + "Mix Preview" label |
| Original Volume slider | Reuse existing `SliderControl` — label "Original", value 0–100%, maps to `original_volume` |
| TTS Volume slider | New `SliderControl` — label "TTS", value 0–100%, maps to `volume` |
| BGM Volume slider | Reuse existing `SliderControl` — label "BGM", value 0–100%, maps to `audio_volume` |
| Mute toggle per track | 3× icon button (`SpeakerHigh` → `SpeakerSlash`) at 20px, uses `ico-btn` pattern, positioned left of each slider |

**Layout:** Vertical stack with `1px` gap (existing `border-subtle` separator pattern). Each slider row uses `.pr` layout (label 60px min-width + flex-1 slider).

### 2. Timeline Mini Mixer (Quick Access)

**Location:** Timeline track headers (`.tllbls` column, `52px` wide).

| Element | Specification |
|---------|--------------|
| Volume indicator | Horizontal bar, 3px height, positioned below track label text |
| Fill width | Proportional to volume (0–100%), uses track color |
| Background | `var(--border)` (unfilled portion) |
| Interaction | Click on indicator → toggle mute (no dragging — too small for precise control) |
| Muted state | Indicator bar opacity → 0.3, track label color → `var(--text-disabled)` |

**Dimensions:** Indicator bar = full width of label area minus 8px padding (each side 4px). Height = 3px. Border-radius = 2px.

### 3. No New Pages or Modals

This phase adds sections to existing panels only — no new routes, dialogs, or popups.

## Interaction Contract

### Volume Slider Behavior

| Action | Response | Latency Target |
|--------|----------|---------------|
| Drag slider thumb | `GainNode.gain.value` updates in real-time | <16ms (requestAnimationFrame) |
| Release slider | Zustand store persists final value | Immediate |
| Click mute icon | `GainNode.gain.value → 0` (mute) or restore previous value (unmute) | <16ms |

### Seek & Scrub Sync (from D-10)

| Action | Audio Behavior |
|--------|---------------|
| Click seek (scrub bar click) | Recalculate TTS segment scheduling, resume all 3 audio sources from new position |
| Continuous scrub (drag) | All audio silent — visual-only scrub |
| Play after scrub | Resume full mix from scrub position |

### AudioContext Lifecycle

| Event | Action |
|-------|--------|
| Tab becomes visible | `AudioContext.resume()` |
| Tab becomes hidden | `AudioContext.suspend()` (preserve battery) |
| Component unmount | Disconnect all source nodes, do NOT close AudioContext (singleton reuse) |
