# Phase 08.3: Main Track Timeline Refactor — Research

## Tổng Quan

Phân tích toàn diện codebase Timeline hiện tại để phục vụ refactor sang kiến trúc Main Track giống CapCut.

## 1. Kiến Trúc Hiện Tại

### Timeline.tsx — 652 dòng monolithic
File trung tâm chứa TẤT CẢ logic timeline trong một component duy nhất:

| Concern | Dòng | Ghi chú |
|---------|------|---------|
| TimelineRuler (memo) | 31-80 | Reusable, có thể extract nguyên vẹn |
| State variables | 87-101 | 15 useState hooks, 3 useRef |
| Duration/zoom calculation | 103-149 | Phụ thuộc vào `useProjectStore` |
| Split/Delete handlers | 151-186 | Hardcode `selectedClipId` patterns: `vid-main`, `audio-main`, `sub-*` |
| Playhead sync | 229-249 | requestAnimationFrame loop, query `document.querySelector('video')` |
| Drag/Resize pointer events | 251-343 | Mega useEffect xử lý cả vid, audio, sub dragging |
| Keyboard shortcuts | 345-383 | Inline keydown handler |
| Track layout (JSX) | 416-651 | 3 track rows hardcode, inline styles ~200 dòng |

**Vấn đề chính:**
- Không có abstraction cho track hoặc clip — tất cả là inline logic
- Track layout hardcoded 3 dòng (VID/SUB/BGM) — không mở rộng được
- Clip IDs dùng string pattern matching (`vid-main`, `audio-main`, `sub-{id}`)
- Drag/resize/snap logic interleaved — rất khó test

### Track Components

| Component | File | Dòng | Chức năng |
|-----------|------|------|-----------|
| VideoTrack | `VideoTrack.tsx` | 108 | Thumbnail blocks virtualized qua IntersectionObserver |
| AudioTrack | `AudioTrack.tsx` | 158 | WaveSurfer chunked rendering, zoom support |
| SubtitleTrack | `SubtitleTrack.tsx` | 158 | Block rendering + drag/trim per segment |

**Nhận xét:**
- VideoTrack/AudioTrack là **pure visual renderers** — đúng vai trò, giữ lại
- SubtitleTrack tự quản lý drag state riêng — sẽ cần adapt cho clip model
- Tất cả đều nhận `pixelsPerSecond`, `scrollLeft`, `viewportWidth` — interface thống nhất

### Stores

#### useProjectStore.ts (334 dòng)
- **91 config fields** trong `ProjectConfig` interface
- Video clip data: `vid_clip_start`, `vid_clip_duration` — **single clip only**
- Audio clip data: `audio_file`, `audio_clip_start`, `audio_clip_duration`, `audio_timeline_start` — **single clip only**
- Timeline state: `timelineZoom`, `snapEnabled`, `snapThreshold`, `selectedClipId`
- `splitLeft()` / `splitRight()` methods — hardcoded cho `vid-main` / `audio-main`
- **Persisted** via `zustand/persist` → cần migration strategy

#### useSubtitleStore.ts (274 dòng)
- `SubtitleSegment` interface: `{ id, start, end, source_text, translated_text, confidence, tts_status, tts_audio_path }`
- Segment-based model (KHÔNG phải clip model)
- Operations: split, merge, trim, delete, ripple delete
- `fileSegments: Record<string, SubtitleSegment[]>` — per-file storage

### Integration Points

| Consumer | File | Cách dùng |

## 2. Khoảng Cách Giữa Hiện Tại → Main Track

### Data Model Gap
| Hiện tại | Cần chuyển sang |
|----------|-----------------|
| `vid_clip_start/duration` (single) | `Clip[]` trên Main Track |
| `audio_*` fields (single BGM) | `Clip[]` trên BGM Track |
| `SubtitleSegment[]` (segment model) | `SubtitleClip[]` (clip model) |
| `selectedClipId: string` (pattern) | `selectedClipId: string` (UUID) |

### Layout Gap
| Hiện tại | Cần chuyển sang |
|----------|-----------------|
| VID / SUB / BGM (3 dòng cố định) | SUB / **MAIN** / TTS / BGM (4 dòng, Main nổi bật) |
| Track headers 70px (label + mute) | Track headers có lock 🔒 + visibility 👁️ + volume 🔊 + menu ⋯ |
| Track height: 50px cố định | Main Track: 60px+, others: 40px |

### Logic Gap
| Hiện tại | Cần chuyển sang |
|----------|-----------------|
| Split/Delete hardcode theo clip type | Generic clip operations |
| Snap: chỉ video edges + audio start | Snap: tất cả clip edges trên mọi track |
| Drag: chỉ reposition vid/audio | Drag: append clip vào track |
| No ripple across tracks | Ripple delete ảnh hưởng tất cả tracks |

## 3. Chiến Lược Refactor

### Approach: Bottom-Up Store → Component

**Wave 1: Data Foundation**
1. Tạo `useTimelineStore.ts` mới — clip registry + track registry
2. Define `Clip`, `Track`, `TrackType` interfaces
3. Migration utility: chuyển `useProjectStore` vid/audio fields → clips
4. SubtitleSegment → SubtitleClip converter

**Wave 2: Component Decomposition**
1. Extract `TimelineToolbar.tsx` từ toolbar section
2. Extract `TrackHeader.tsx` — track label + control panel (lock/visibility/volume/menu)
3. Extract `TrackRow.tsx` — generic clip container per track
4. Tạo `ClipBlock.tsx` — generic clip visual renderer
5. Refactor `Timeline.tsx` → orchestrator component (layout + scroll + playhead)

**Wave 3: Behavior Migration**
1. Move drag/resize handlers vào `useTimelineDrag.ts` hook
2. Update snap logic cho multi-clip
3. Wire playhead sync
4. Wire keyboard shortcuts cho new clip model
5. Drag-drop từ Media Bin vào Main Track (append)

### Rủi Ro

| Rủi ro | Mức | Giải pháp |
|--------|-----|-----------|
| Break existing subtitle editing | Cao | SubtitleClip adapter giữ nguyên API surface |
| Persist migration (zustand) | Trung bình | Version field + migration function |
| AudioMixer dependency trên segments | Trung bình | Adapter `clips → segments` cho AudioMixer |
| Performance regression (multi-clip render) | Thấp | Giữ virtualization pattern hiện tại |

## 4. Validation Architecture

### Dimension 1: Functional Correctness
- Main Track hiển thị multi-clip nối tiếp
- Drag video từ Media Bin → append lên Main Track
- Subtitle vẫn hoạt động (drag, trim, split, delete)
- BGM track vẫn hoạt động (drag, resize, delete)

### Dimension 2: Visual Correctness
- Track layout: SUB / MAIN / TTS / BGM (đúng thứ tự)
- Main Track nổi bật hơn (height lớn hơn, border accent)
- Track headers có control panel đầy đủ

### Dimension 3: Integration
- Playhead sync không bị break
- AudioMixer vẫn hoạt động với clip data
- Keyboard shortcuts hoạt động
- Snap logic hoạt động với multi-clip edges

### Dimension 4: Regression
- Video preview vẫn play đúng
- Export/Render pipeline vẫn nhận đúng data
- Project persistence (zustand) load/save đúng
